6.2.3.7. B: [Step by Step] Compiling a Component¶

How to compile your component exactly depends hardly on the Platform/toolchain you are using. We cannot describe every IDE or Makefile mechanism here, but we picked up a few examples.

  • Create a new project to compile the source files of our new component:

    1. VxWorks / Tornado:

      1. File -> New Project

      2. Select „Create downloadable application modules for VxWorks“

      3. Finish the Wizard

      4. Right click on new Project -> Add files …

      5. Select your Source file (CmpTest.c)

    2. VxWorks / Workbench:

      1. File -> New -> VxWorks Downloadable Kernel Module Project

      2. Finish the Wizard

      3. Delete second build-target named <project-name>_partialImage

      4. Delete reference to <project-name>_partialImage from first build target

      5. Copy your Component folder under your newly created project.

      6. Right click on the project -> Refresh

      7. Right click on the build target -> Edit content

      8. Select your source file (CmpTest.c) and press „Add“

      9. Right click on the project -> Build Options -> Set Active Build Spec…

      10. Select the corresponding build specification for your platforms
        (Note, that we only support the GNU compiler)

    3. Visual Studio 2015:

      1. File -> New -> Project

      2. Visual C++ -> Win32 and here select a “Win32 Project”

      3. Select a name and the path for the project and solution

      4. Select in the Wizard “Application Settings” and select here “DLL”

      5. Check “Empty project” and uncheck “Precompiled header”

      6. Uncheck optional “Security Development Lifecycle (SDL) checks” if not necessary

      7. Finish the Wizard

      8. Change to „Solution Explorer“ Tab

      9. Right click on „Source Files“ -> Add -> Existing Item…

      10. Select you source file (CmpTest.c)

      11. Right click on „Header Files“ -> Add -> Existing Item…

      12. Select your header files (*Itf.h and *Dep.h) and both M4 files

      13. Right click on each m4 file and select Properties

      14. Go to category General and select in “Excluded From Build” the option “Yes”

      15. Close the property dialog with saving the changes

    4. Linux:

      1. Use the supported makefile under Templates/CmpTemplate/Linux as a basis for your component

  • Add Compiler Defines and Include Paths:
    The most essential preprocessor defines are:
    STATIC_LINK, DYNAMIC_LINK and for some RISC CPUs also NO_PRAGMA_PACK

    1. VxWorks / Tornado:

      1. Change to the „Builds“ Tab

      2. Right click on the „Build Spec“ under your project -> Properties

      3. Change to the „C/C++ Compiler“ Tab

      4. Add the following compiler flags:

        1. -DMIXED_LINK (that’s always necessary in VxWorks)

        2. -I<path-to-runtime-interfaces>/Components

        3. -I<path-to-runtime-interfaces>/Platforms/VxWorks

    2. VxWorks / Workbench:

      1. Right click on build target -> Properties

      2. Change to „Build Macros“ Tab

      3. Add the following Define:

        1. -DMIXED_LINK (that’s always necessary in VxWorks)

      4. Change to „Build Paths“ Tab

      5. Add the following Build Paths to your configuration:

        1. -I<path-to-runtime-interfaces>/Components

        2. -I<path-to-runtime-interfaces>/Platforms/VxWorks

    3. Visual Studio 2015:

      1. Project -> Properties

      2. Change to „C/C++“ Category

      3. Select Category „General“

      4. Add the following paths, comma separated to „Additional

        Include Directories“:

        1. <path-to-runtime-interfaces>/Components

        2. <path-to-runtime-interfaces>/Platforms/Windows

    4. Linux:

    1. Additional hints:

      • The template makefile requires the tools “m4” and “to/fromdos” installed to work properly, usually they are included in your Linux derivate, or at least installable via the default package mechanism.

    2. Adopt and check the following variables in the makefile:

    • TARGET: should be the component name (without lib prefix and without “.so” extension): “CmpTemplate” is the default (this leads to an object called “libCmpTemplate.so”).

    • INCLUDE: should include all required include paths for compilation

    • CXXFLAGS

      –> TRG_ define: at least one “TRG_” define is required: choose the correct one from “TRG_X86 | TRG_PPC | TRG_ARM | TRG_CORTEX”.
