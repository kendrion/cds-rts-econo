5.2.130.5. Detailed Description¶

Interface of SysGraphicTestable.

The interface SysGraphicTestable is intended for implementing automatic testability in a graphical component. Typically this interface can be implemented by components intended for embedded graphics (framebuffer, light visualization). It is primarily meant as a simplification of SysGraphicNative to support automated testing.

5.2.130.8.1. RTS_HANDLE SysGraphicTestableCreate (RTS_RESULT *pResult)¶

Creates a handle to a test object that can be used in further calls. If you do not use this handle any more, release it using SysGraphicTestableDestroy

Parameters

[out] pResult Optionally returns an error code For Implementers: In fact nothing has to be created; it’s merely a way for getting calls to initialize and shutdown.

Returns:

The handle to the test object or RTS_INVALID_HANDLE in case of errors.

5.2.130.8.2. RTS_RESULT SysGraphicTestableDestroy (RTS_HANDLE hTestable)¶

Releases a handle to a test object that was returned by SysGraphicTestableCreate.

Parameters

[in] hDisplayContext The native displaycontext to release

Returns:

An error code

5.2.130.8.3. RTS_RESULT SysGraphicTestableGetDisplaySize (RTS_HANDLE hTestable, RTS_Size *pSize)¶

Returns the size of the display.

[out] pSize This structure will be filled with the size of the native display.

Returns:

5.2.130.8.4. RTS_RESULT SysGraphicTestableGetDisplayPixel (RTS_HANDLE hTestable, short sX, short sY, unsigned long *pulColorValue)¶

This method can be called for retrieving the color of a displayed pixel.

Parameters

[in] hDC The testable object
[in] sX The x coordinate to get the color from
[in] sY The y coordinate to get the color from
[in] pulColorValue Pointer that will return the determined color. The color will be in the format used by the iec visualization ([ |RRRRRRRR|GGGGGGGG|BBBBBBBB]).

Returns:

An error code
