Macro Definition Documentation¶

#define CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_ADDRESS “Communication.AddressDest”¶

The address this remote client should connect to. When an explicit communication address is assigned, then the setting CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_PLCNAME and CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_TCP_ADDRESS will be ignored.

#define CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_ADDRESS_DEFAULT “”¶

#define CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_PLCNAME “Communication.PlcNameDest”¶

The name of the plc this remote client should connect to If this mode is used, then a network scan will be used for finding the according device. When an explicit communication address is assigned in CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_ADDRESS, then this setting will be ignored. To be able to use this feature, the component CmpNameServiceClient must be available!

#define CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_PLCNAME_DEFAULT “”¶

#define CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_PLCNAMEA “Communication.PlcNameDestA”¶

This setting is similar than CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_PLCNAME with the difference that it allows an ASCII value for the plc name. If CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_PLCNAME is given too, that one will be preferred.

#define CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_PLCNAMEA_DEFAULT CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_PLCNAME_DEFAULT¶

#define CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_TCP_ADDRESS “Communication.TcpAddressDest”¶

The ip address this remote client should connect to using CmpBlkDrvTcp. If this type of communication address is configured, then the port can be configured using CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_TCP_ADDRESS_PORT too if it differs from the default port.

#define CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_TCP_ADDRESS_DEFAULT “”¶

#define CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_TCP_ADDRESS_PORT “Communication.TcpAddressDestPort”¶

The tcp port this remote client should connect to using CmpBlkDrvTcp. Will be evaluated only when CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_TCP_ADDRESS is configured. If nothing is configured, in fact the default value -1, then the default port of CmpBlkDrvTcp will be used.

#define CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_TCP_ADDRESS_PORT_DEFAULT -1¶

#define CMPVISUHANDLERREMOTE_KEY_VISUCOMMBUFFER_SIZE “Communication.BufferSize”¶

The size of the communication buffer that should be used.

#define CMPVISUHANDLERREMOTE_KEY_VISUCOMMBUFFER_SIZE_DEFAULT INT32_C(50000)¶

#define CMPVISUHANDLERREMOTE_KEY_VISUCOMMTASK_INTERVAL “Communication.TaskInterval”¶

The size of the communication buffer that should be used.

#define CMPVISUHANDLERREMOTE_KEY_VISUCOMMTASK_INTERVAL_DEFAULT 50¶

#define CMPVISUHANDLERREMOTE_KEY_VISUCOMMTASK_PRIORITY “Communication.TaskPriority”¶

The size of the communication buffer that should be used.

#define CMPVISUHANDLERREMOTE_KEY_VISUCOMMTASK_PRIORITY_DEFAULT 80¶

#define CMPVISUHANDLERREMOTE_KEY_VISUCOMM_RECONNECT_WAITTIME “Communication.ReconnectTimeMs”¶

The time that will be waited before a reconnect is done (in case of a communication error etc.). If a value of 0 is given, there won’t be a reconnect. The application will then shutdown in case of an error.

#define CMPVISUHANDLERREMOTE_KEY_VISUCOMM_RECONNECT_WAITTIME_DEFAULT INT32_C(5000)¶

#define CMPVISUHANDLERREMOTE_KEY_VISUCOMM_NETWORKSCAN_TIME “Communication.NetworkScanTimeMs”¶

The time that will be waited before for the result of a network scan (in case CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_PLCNAME is used).

#define CMPVISUHANDLERREMOTE_KEY_VISUCOMM_NETWORKSCAN_TIME_DEFAULT INT32_C(2000)¶

#define CMPVISUHANDLERREMOTE_KEY_VISUCOMM_SERVICE_TIMEOUT “Communication.ServiceTimeoutMs”¶

The maximum time that will be waited for the answer to a service request that is sent to the plc in milliseconds. A negative value will be an infinite timeout.

#define CMPVISUHANDLERREMOTE_KEY_VISUCOMM_SERVICE_TIMEOUT_DEFAULT INT32_C(-1)¶

#define CMPVISUHANDLERREMOTE_KEY_VISUCOMM_EXECUTE_APPLOGIN “Communication.ExecuteApplicationLogin”¶

This setting specifies whether an application login is done during connecting to the plc. This login is necessary to check for a currently executing download that might take a longer time to prevent followup issues like missing images. In case this application login is not possible (maybe due to accessrights) and therefore should be omitted at all, this value can be set to 0.

#define CMPVISUHANDLERREMOTE_KEY_VISUCOMM_EXECUTE_APPLOGIN_DEFAULT INT32_C(1)¶

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_PAINTBUFFER_SIZE “VisuClient.PaintBufferSize”¶

The size of the paintbuffer for the visualization that should be used. This size should be at least the size of the communication buffer

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_PAINTBUFFER_SIZE_DEFAULT INT32_C(50000)¶

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_VISUAPPNAME “VisuClient.VisuAppName”¶

The name of the visu application on the plc.

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_VISUAPPNAME_DEFAULT “”¶

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_STARTVISU “VisuClient.StartVisu”¶

The name of the startvisualization that should be displayed.

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_STARTVISU_DEFAULT “PLC_VISU”¶

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_ANTIALIASING “VisuClient.AntiAliasing”¶

The setting for antialiasing. Currently supported are (depending on the platform) 2 for high quality, 1 for low quality and 0 for no antialiasing

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_ANTIALIASING_DEFAULT 2¶

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_BESTFIT “VisuClient.BestFit”¶

The setting for bestfit. Currently supported are 1 for activate bestfit mode and 0 for deactivated best fit.

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_BESTFIT_DEFAULT 0¶

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_BESTFITFORDIALOGS “VisuClient.BestFitForDialogs”¶

The setting bestfit for dialogs. Currently supported are 1 for activate bestfit for dialogs mode and 0 for deactivated best fit for dialogs. If this setting is set dialogs are scaled in the same way as top level visualizations

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_BESTFITFORDIALOGS_DEFAULT 0¶

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_SCALETYPEISOTROPIC “VisuClient.ScaleTypeIsotropic”¶

The setting for scale type isotropic in best fit mode. Currently supported are 1 for activate scale type isotropic mode and 0 for deactivated scale type isotropic mode.

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_SCALETYPEISOTROPIC_DEFAULT 0¶

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_TOUCHHANDLING “VisuClient.Touchhandling”¶

The setting for touchhandling. Currently supported are 1 for activate touchhandling mode and 0.

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_TOUCHHANDLING_DEFAULT 0¶

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_KEYBOARD “VisuClient.Keyboard”¶

The setting for keyboard useage. Currently supported are 1 for activating keyboard usage and 0.

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_KEYBOARD_DEFAULT 0¶

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_BENCHMARKING “VisuClient.Benchmarking”¶

The setting for benchmarking. Currently supported are 1 to activate benchmarking measurements 0 (off, default). The intention of this setting is to measure execution time of the visualization. It is not intended for use in production environments.

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_BENCHMARKING_DEFAULT 0¶

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_IMAGE_LOAD_ATTEMPTS “VisuClient.ImageLoadAttempts”¶

The setting for the number of attempts to load an image.

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_IMAGE_LOAD_ATTEMPTS_DEFAULT 3¶

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_CLIENTNAME “VisuClient.Name”¶

An optional name for the visualization name. This name can optionally be used to distinguish between different clients within the application. Default value is the empty string.

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_CLIENTNAME_DEFAULT “”¶

#define CMPVISUHANDLERREMOTE_KEY_VISUCREDENTIALS_USERNAME “Credentials.UserName”¶

The name of the user used for logging in to the remote plc.

#define CMPVISUHANDLERREMOTE_KEY_VISUCREDENTIALS_USERNAME_DEFAULT “”¶

#define CMPVISUHANDLERREMOTE_KEY_VISUCREDENTIALS_PASSWORD “Credentials.Password”¶

The password of the user used for logging in to the remote plc. This setting is the way for manual configuration of the password.

#define CMPVISUHANDLERREMOTE_KEY_VISUCREDENTIALS_PASSWORD_DEFAULT “”¶

#define CMPVISUHANDLERREMOTE_KEY_VISUCREDENTIALS_PASSWORD_OBFUSCATED “Credentials.Obfuscated”¶

The obfuscated value of the user’s password.

As the application must be able to decrypt the password to send it to the connected plc, we can use only a symmetric encryption algorithm. For that reason, a cryptographically secure storage of the password is not possible this way. This is merely the prevention of users having a chance of looking at the configuration file to see the password in cleartext. This setting will be created automatically during startup of the remote targetvisualization if CMPVISUHANDLERREMOTE_KEY_VISUCREDENTIALS_PASSWORD

is found in the configuration. For that reason this is not a value that is intended to be written by the user.

#define CMPVISUHANDLERREMOTE_VISUCREDENTIALS_PASSWORD_OBFUSCATED_DEFAULT “”¶

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_NOSYSTRAY “VisuClient.NoSysTray”¶

This setting is relevant for platforms, that show a systray icon, that allows access to the remote target visualization, e.g. to display the error messages. The values have the following meaning:

  • 0: Systray icon will be used
  • 1: Systray icon will be not used, a window will be displayed instead of

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_NOSYSTRAY_DEFAULT 0¶

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_FORCEWIZARD “VisuClient.ForceWizard”¶

This setting defines the behaviour, whether the wizard to configure the communication settings should be displayed or not. The values have the following meaning:

  • 0: Wizard will be only shown if the configuration file is missing or contains no communication parameters (the default!)
  • 1: Wizard will be displayed and the communication settings can be saved persistently in the configuration file
  • 2: Wizard will be displayed and the communication settings will be not saved persistently in the configuration file, so that the wizard will be displayed again in the next program run. This setting is effective only when an according wizard functionality is implemented in the system specific part

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_FORCEWIZARD_DEFAULT 0¶

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_DEVICEID “VisuClient.DeviceId”¶

This setting defines the behaviour of the device scan of the wizard to configure the communication settings. The values have the following meaning:

  • 0: No device ID specified. All found devices will be listed in the wizard (the default!) all other positive values: Will be used as filter criterion for the device type.

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_DEVICEID_DEFAULT 0¶

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_DIRECTCONNECTION “VisuClient.DirectConnection”¶

This is the prefix of the keys, that are used to store the last used direct connections persistently. Up to CMPVISUHANDLERREMOTE_DIRECTCONNECTION_MAXENTRIES such entries will be stored.

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_DIRECTCONNECTION_DEFAULT “”¶

#define CMPVISUHANDLERREMOTE_DIRECTCONNECTION_MAXENTRIES 8¶
