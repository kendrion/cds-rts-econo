Macro Definition Documentation¶

#define CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_ADDRESS “Communication.AddressDest”¶

The address this remote client should connect to. When an explicit communication address is assigned, then the setting CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_PLCNAME and CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_TCP_ADDRESS will be ignored.

#define CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_ADDRESS_DEFAULT “”¶

#define CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_PLCNAME “Communication.PlcNameDest”¶

The name of the PLC this remote client should connect to If this mode is used, then a network scan will be used for finding the according device. When an explicit communication address is assigned in CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_ADDRESS, then this setting will be ignored. To be able to use this feature, the component CmpNameServiceClient must be available!

#define CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_PLCNAME_DEFAULT “”¶

#define CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_PLCNAMEA “Communication.PlcNameDestA”¶

This setting is similar than CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_PLCNAME with the difference that it allows an ASCII value for the PLC name. If CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_PLCNAME is given too, that one will be preferred.

#define CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_PLCNAMEA_DEFAULT CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_PLCNAME_DEFAULT¶

#define CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_TCP_ADDRESS “Communication.TcpAddressDest”¶

The IP address this remote client should connect to using CmpBlkDrvTcp. If this type of communication address is configured, then the port can be configured using CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_TCP_ADDRESS_PORT too if it differs from the default port.

#define CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_TCP_ADDRESS_DEFAULT “”¶

#define CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_TCP_ADDRESS_PORT “Communication.TcpAddressDestPort”¶

The TCP port this remote client should connect to using CmpBlkDrvTcp. Will be evaluated only when CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_TCP_ADDRESS is configured. If nothing is configured, in fact the default value -1, then the default port of CmpBlkDrvTcp will be used.

#define CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_TCP_ADDRESS_PORT_DEFAULT -1¶

The size of the communication buffer that should be used.

The size of the communication buffer that should be used.

#define CMPVISUHANDLERREMOTE_KEY_VISUCOMM_RECONNECT_WAITTIME “Communication.ReconnectTimeMs”¶

The time that will be waited before a reconnect is done (in case of a communication error etc.). If a value of 0 is given, there won’t be a reconnect. The application will then shutdown in case of an error.

#define CMPVISUHANDLERREMOTE_KEY_VISUCOMM_SERVICE_TIMEOUT “Communication.ServiceTimeoutMs”¶

The maximum time that will be waited for the answer to a service request that is sent to the PLC in milliseconds. A negative value will be an infinite timeout.

#define CMPVISUHANDLERREMOTE_KEY_VISUCOMM_SERVICE_TIMEOUT_DEFAULT INT32_C(-1)¶

#define CMPVISUHANDLERREMOTE_KEY_VISUCOMM_EXECUTE_APPLOGIN “Communication.ExecuteApplicationLogin”¶

This setting specifies whether an application login is done during connecting to the PLC. This login is necessary to check for a currently executing download that might take a longer time to prevent follow up issues like missing images. In case this application login is not possible (maybe due to access rights) and therefore should be omitted at all, this value can be set to 0.

#define CMPVISUHANDLERREMOTE_KEY_VISUCOMM_EXECUTE_APPLOGIN_DEFAULT INT32_C(1)¶

#define CMPVISUHANDLERREMOTE_KEY_VISUCOMM_ENCRYPTION_MODE “Communication.EncryptionMode”¶

This setting specifies the way the remote targetvisu communicates with the PLC. At the moment only the following two setting values are supported: 0: Don’t use encrypted communication. 1: Communicate in an encrypted way. This means that the operator either has to install certificates of the PLC or accept the certificate during communication setup.

#define CMPVISUHANDLERREMOTE_KEY_VISUCOMM_ENCRYPTION_MODE_DEFAULT INT32_C(0)¶

#define CMPVISUHANDLERREMOTE_KEY_VISUCOMM_ENCRYPTION_CIPHERS “Communication.EncryptionCipherSuite”¶

<type>String</type>

This setting allows to override the ciphersuite that should be used for encrypted communication. For details of the syntax please refer to the description of the setting TLSKEY_STR_CTX_CIPHER_LIST

#define CMPVISUHANDLERREMOTE_KEY_VISUCREDENTIALS_PASSWORD “Credentials.Password”¶

The password of the user used for logging in to the remote PLC. This setting is the way for manual configuration of the password.

#define CMPVISUHANDLERREMOTE_KEY_VISUCREDENTIALS_PASSWORD_DEFAULT “”¶

#define CMPVISUHANDLERREMOTE_KEY_VISUCREDENTIALS_PASSWORD_OBFUSCATED “Credentials.Obfuscated”¶

The obfuscated value of the user’s password.

As the application must be able to decrypt the password to send it to the connected PLC, we can use only a symmetric encryption algorithm. For that reason, a cryptographically secure storage of the password is not possible this way. This is merely the prevention of users having a chance of looking at the configuration file to see the password in clear-text. This setting will be created automatically during startup of the remote target-visualization if CMPVISUHANDLERREMOTE_KEY_VISUCREDENTIALS_PASSWORD

is found in the configuration. For that reason this is not a value that is intended to be written by the user.

#define CMPVISUHANDLERREMOTE_VISUCREDENTIALS_PASSWORD_OBFUSCATED_DEFAULT “”¶

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_NOSYSTRAY “VisuClient.NoSysTray”¶

This setting is relevant for platforms, that show a system tray icon, that allows access to the remote target visualization, e.g. to display the error messages. The values have the following meaning:

  • 0: System tray icon will be used

  • 1: System tray icon will be not used, a window will be displayed instead of

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_NOSYSTRAY_DEFAULT 0¶

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_FORCEWIZARD “VisuClient.ForceWizard”¶

This setting defines the behavior, whether the wizard to configure the communication settings should be displayed or not. The values have the following meaning:

  • 0: Wizard will be only shown if the configuration file is missing or contains no communication parameters (the default!)

  • 1: Wizard will be displayed and the communication settings can be saved persistently in the configuration file

  • 2: Wizard will be displayed and the communication settings will be not saved persistently in the configuration file, so that the wizard will be displayed again in the next program run.

This setting is effective only when an according wizard functionality is implemented in the system specific part

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_FORCEWIZARD_DEFAULT 0¶

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_DEVICEID “VisuClient.DeviceId”¶

This setting defines the behavior of the device scan of the wizard to configure the communication settings. The values have the following meaning:

  • 0: No device ID specified. All found devices will be listed in the wizard (the default!) all other positive values: Will be used as filter criterion for the device type.

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_DEVICEID_DEFAULT 0¶

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_DIRECTCONNECTION “VisuClient.DirectConnection”¶

This is the prefix of the keys, that are used to store the last used direct connections persistently. Up to CMPVISUHANDLERREMOTE_DIRECTCONNECTION_MAXENTRIES such entries will be stored.

#define CMPVISUHANDLERREMOTE_KEY_VISUCLIENT_DIRECTCONNECTION_DEFAULT “”¶
