Macro Definition Documentation¶

#define APPKEY_INT_CREATE_BOOTPROJECT_ON_DOWNLOAD “Bootproject.CreateOnDownload”¶

<type>Int</type>

Setting to enable/disable creating boot application implicit on download: 1: Create boot application implicit 0: Don’t create boot application implicit

#define APPVALUE_INT_CREATE_BOOTPROJECT_ON_DOWNLOAD_DEFAULT 1¶

#define APPKEY_INT_STORE_BOOTPROJECT_ONLY_ON_DOWNLOAD “Bootproject.StoreOnlyOnDownload”¶

<type>Int</type>

Setting to enable/disable only storing boot application implicit on download without storing in RAM: 1: Store boot application only on download 0: Don’t store boot application only on download

#define APPVALUE_INT_STORE_BOOTPROJECT_ONLY_ON_DOWNLOAD_DEFAULT 0¶

#define APPKEY_INT_BOOTPROJECT_INVALIDATE_BY_RENAME “Bootproject.InvalidateByRename”¶

<type>Int</type>

Setting to invalidate boot application by rename. If not, the key APPKEY_INT_LOAD_BOOTPROJECT can be used to invalidate the boot application.

#define APPKEY_INT_BOOTPROJECT_INVALIDATE_BY_RENAME_DEFAULT 0¶

#define APPKEY_INT_BOOTPROJECT_INVALIDATE_BY_SETTING “Bootproject.InvalidateBySetting”¶

<type>Int</type>

Setting to invalidate boot application by setting (key APPKEY_INT_LOAD_BOOTPROJECT)

#define APPKEY_INT_BOOTPROJECT_INVALIDATE_BY_SETTING_DEFAULT 0¶

#define APPKEY_INT_BOOTPROJECT_INVALIDATE_NEVER “Bootproject.InvalidateNever”¶

<type>Int</type>

Setting to disable invalidating boot application (1=disables both APPKEY_INT_BOOTPROJECT_INVALIDATE_BY_RENAME and APPKEY_INT_BOOTPROJECT_INVALIDATE_BY_SETTING)

#define APPKEY_INT_BOOTPROJECT_INVALIDATE_NEVER_DEFAULT 1¶

#define APPKEY_INT_BOOTPROJECT_RUN_IN_FLASH “Bootproject.RunInFlash”¶

<type>Int</type>

Setting to invalidate boot application by rename. If not, the next key (APPKEY_INT_LOAD_BOOTPROJECT) can be used to invalidate the boot application.

#define APPKEY_INT_BOOTPROJECT_RUN_IN_FLASH_DEFAULT 0¶

#define APPKEY_INT_LOAD_BOOTPROJECT “.Load”¶

<type>Int</type>

Setting to disable the boot application. The application name must be used as prefix for this key!

#define APPKEY_INT_LOAD_BOOTPROJECT_DEFAULT 1¶

#define APPKEY_INT_START_BOOTPROJECT “.Start”¶

<type>Int</type>

Setting to specify whether the boot application should start on startup or not. The application name must be used as prefix for this key: e.g. Application.Start=0

#define APPKEY_INT_START_BOOTPROJECT_DEFAULT 1¶

#define APPKEY_INT_LOCKEDSTATE “.LockedState”¶

<type>Int</type>

Setting to restore the state of an application if the device is operating in locked mode. The application name must be used as prefix for this key: e.g. Application.LockedState=1 (AS_RUN)

#define APPKEY_INT_LOCKEDSTATE_DEFAULT 0¶

#define APPKEY_INT_FIRST_LOAD_INIT_RETAINS “.FirstLoadInitRetains”¶

<type>Int</type>

Setting to initialize the retains during the first load of an offline boot application. This key is automatically removed after the first load. The application name must be used as prefix for this key!

#define APPKEY_INT_FIRST_LOAD_INIT_RETAINS_DEFAULT 0¶

#define APPKEY_STRING_BOOTPROJECT_NAME “.Bootproject”¶

<type>String</type>

Setting to set the boot application of an application. The application name must be used as prefix for this key: e.g. Application.Bootproject=App1

#define APPKEY_STRING_RETAIN_TYPE_APPLICATIONS “RetainType.Applications”¶

<type>String</type>

Settings to set the retain management

#define APPVALUE_STRING_RETAIN_TYPE_ONPOWERFAIL “OnPowerfail”¶

#define APPVALUE_STRING_RETAIN_TYPE_INSRAM “InSRAM”¶

#define APPVALUE_STRING_RETAIN_TYPE_NONE “None”¶

#define APPVALUE_STRING_RETAIN_TYPE_DEFAULT APPVALUE_STRING_RETAIN_TYPE_ONPOWERFAIL¶

#define APPKEY_INT_LOAD_WITH_RETAIN_MISMATCH “Bootproject.LoadWithRetainMismatch”¶

<type>Int</type>

OBSOLETE SETTING: “Bootproject.RetainMismatch.Init” should be used instead. Only for backward compatibility.

#define APPVALUE_INT_LOAD_WITH_RETAIN_MISMATCH_DEFAULT 0¶

#define APPKEY_INT_BOOTPROJECT_RETAIN_MISMATCH_INIT “Bootproject.RetainMismatch.Init”¶

<type>Int</type>

Setting to define the behavior on loading boot project, if retains do not match. In this case, all retains are initialized and the boot application will be started! Both keys are equivalent!

#define APPVALUE_INT_BOOTPROJECT_RETAIN_MISMATCH_INIT_DEFAULT 0¶

#define APPKEY_INT_BOOTPROJECT_RETAIN_MISMATCH_EXCEPTION “Bootproject.RetainMismatch.Exception”¶

<type>Int</type>

Setting to define the behavior on loading boot project, if retains do not match. In this case, the boot application will remain in stop and in exception state! Only a manual reset can heal this state!

#define APPVALUE_INT_BOOTPROJECT_RETAIN_MISMATCH_EXCEPTION_DEFAULT 0¶

#define APPKEY_INT_ASYNC_SERVICES “AsyncServices”¶

<type>Int</type>

Handle all application online services asynchronously.

#define APPVALUE_INT_ASYNC_SERVICES_DEFAULT 0¶

#define APPKEY_INT_WAIT_FOR_TASK_GAP “WaitForTaskGapTimeout”¶

<type>Int</type>

Timeout to wait for a task gap. e.g. to wait for the gap to execute the online-change code.

#define APPVALUE_INT_WAIT_FOR_TASK_GAP_DEFAULT INT32_C(10000)¶

#define APPKEY_INT_GLOBAL_STOP_ON_HARDWARE_EXCEPTION “Exception.Hardware.GlobalStop”¶

<type>Int</type>

If this entry is set, every hardware exception leads to a stop of all applications!

#define APPVALUE_INT_GLOBAL_STOP_ON_HARDWARE_EXCEPTION_DEFAULT 0¶

#define APPKEY_INT_RUN_STOP_TIMEOUT “RunStopTimeoutMs”¶

<type>Int</type>

Default timeout for the RUN-STOP transition of an application. This is the timeout that is controlled by the optional CmpSupervisor component! The value is in milliseconds.

#define APPVALUE_INT_RUN_STOP_TIMEOUT_DEFAULT INT32_C(10000)¶
