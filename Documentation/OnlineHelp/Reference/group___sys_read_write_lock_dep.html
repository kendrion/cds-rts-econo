Detailed Description¶

System component that offers functionality for so called ReadWriteLocks. This is a synchronization object, where you can separate a read access (ReadLock) from a write access (WriteLock) on the same resource.

NOTES:

  • Multiple readers can have access to the resource at the same time. The writer is dominant and have exclusive access to the resource!

  • A write lock cannot be entered, if still a read lock is hold!

  • A read lock cannot be entered, if still a write lock is hold!
