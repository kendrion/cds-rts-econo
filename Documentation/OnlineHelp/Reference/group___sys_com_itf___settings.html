Macro Definition Documentation¶

#define SYSCOMKEY_INT_WINDOWS_PORT_TO_CHECK “Windows.PortToCheck”¶

Allows to specify a serial port by its Microsoft Windows friendly name instead of the COM port number. The friendly name of the COM port is for example shown by the Microsoft Windows Device Manager. This feature is useful, if the port number is somehow dynamic, but if the adapter has a well defined friendly name. Typically it’s used for USB devices, which provide a virtual COM port driver. If the (by SysComOpen) requested COM port number is found in the list of the configured “Windows.PortToCheck” settings, then the port is opened, which matches to the name specified by “Windows.CheckWithPortName”, instead of the requested port number. (Only for Windows). Example: Windows.PortToCheck.0=99 Windows.CheckWithPortName.0=My USB serial device Windows.PortToCheck.1=98 Windows.CheckWithPortName.1=PLC USB interface

#define SYSTIMERVALUE_INT_WINDOWS_PORT_TO_CHECK_DEFAULT 0¶

#define SYSCOMKEY_STRING_WINDOWS_PORTNAME_CHECK_WITH “Windows.CheckWithPortname”¶

Specifies a string, which is compared to the Microsoft Windows friendly name of the available COM ports. The name matches, if it starts with the provided string. See also setting PortToCheck. (Only for Windows).

#define SYSTIMERVALUE_STRING_WINDOWS_PORTNAME_CHECK_WITH_DEFAULT “”¶

#define SYSCOMKEY_STRING_LINUX_DEVICEFILE “Linux.Devicefile”¶

Devicefile for serial interface (Only for Linux).

#define SYSCOMKEY_STRING_LINUX_DEVICEFILE_DEFAULT “/dev/ttyS”¶

#define SYSCOMKEY_INT_LINUX_EXCLUSIVE_ACCESS “Linux.ExclusiveAccess”¶

Activate exclusive access to the devicefile (Only for Linux).

#define SYSCOMKEY_INT_LINUX_EXCLUSIVE_ACCESS_DEFAULT 0¶

#define SYSCOMKEY_STRING_LINUX_PHYSICAL_LAYER “Linux.PhysicalLayer”¶

Physical layer of the serial interface. Valid values are “rs232” and “rs485”. (Only for Linux).

#define SYSCOMKEY_STRING_LINUX_PHYSICAL_LAYER_DEFAULT “rs232”¶

#define SYSCOMKEY_STRING_LINUX_RS485_RTS_PIN “Linux.rs485.RTSPin”¶

on_send: RTS pin is high on send, and low all other times. after_send: RTS pin is low on send. (Only for Linux).

#define SYSCOMKEY_STRING_LINUX_RS485_RTS_PIN_DEFAULT “on_send”¶
