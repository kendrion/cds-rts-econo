Macro Definition Documentation¶

#define VLF_MOTOROLA_BYTE_ORDER 0x00000001¶

Motorola byte order (big endian)

#define VLF_CONSISTENT_BACKGROUND_READ 0x00000002¶

Communication task tries to read the variables in the background. If it is not possible to read all variables without interruption of the IEC tasks during the consistency timeout the read request is denied with an error result ERR_NO_CONSISTENCY!

#define VLF_CONSISTENT VLF_CONSISTENT_BACKGROUND_READ¶

#define VLF_DATA_CHANGE 0x00000004¶

Only variables, which values changed, are transmitted

#define VLF_EXTENDABLE 0x00000008¶

Client requests to work with variable lists, which can extend the comm buffer

#define VLF_EXTENDABLE_SUPPORTED 0x00000010¶

PLC supports variable lists, which can extend the comm buffer

#define VLF_ACCESS_BY_VARHANDLE 0x00000020¶

Client accesses not the complete list, but only the specified vars of the list

#define VLF_SPLIT_HUGE_VARIABLES 0x00000040¶

Client can also read variables, which are larger than the communication buffer

#define VLF_CONSISTENT_SYNCHRONIZED_READ 0x00000080¶

Communication task waits for an IEC task gap and blocks the IEC tasks until the values are copied from the IEC variables to the cache

#define VLF_CONSISTENT_SYNCHRONIZED_WRITE 0x00000100¶

Communication task waits for an IEC task gap and blocks the IEC tasks until the values are copied from the cache into the IEC variables

#define VLF_CONSISTENCY_SUPPORTED 0x00000200¶

PLC supports the requested consistency (VLF_CONSISTENT_BACKGROUND_READ, VLF_CONSISTENT_SYNCHRONIZED_READ or VLF_CONSISTENT_SYNCHRONIZED_WRITE)

#define VLF_EXECUTE_CONSISTENT_READ 0x00000400¶

Flag to mark the first read service of a variable list to read in all values consistent in the cache

#define VLF_EXECUTE_CONSISTENT_WRITE 0x00000800¶

Flag to mark the last write service of a variable list to write out all values consistent from the cache into the IEC variables

#define VLF_RETURN_VARINFO 0x00001000¶

Return complete VarInfo to client, to let the client retrieve the variable the next time by variable information instead of name

#define VLF_TIMESTAMP_UTC 0x00010000¶

Time stamp in UTC (Universal Time Coordinated), at which the values are read

#define VLF_TIMESTAMP_TIMEDATE 0x00020000¶

#define VLF_TIMESTAMP_MS 0x00040000¶

Time stamp in millisecond, at which the values are read

#define VLF_TIMESTAMP_US 0x00080000¶

Time stamp in microseconds, at which the values are read

#define VLF_TIMESTAMP_NS 0x00100000¶

Time stamp in nanoseconds, at which the values are read
