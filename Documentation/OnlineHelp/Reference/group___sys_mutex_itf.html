Interface of SysMutex.

Detailed Description¶

The SysMutex interface is projected to handle synchronization objects for tasks and threads. The synchronization objects are called mutex to synchronize concurrent access to single data resources.

For example:

Task1: … SysMutexEnter(hMutex); [Here you can accessed the protected data] SysMutexLeave(hMutex); [After SysSemLeave() you must not access the protected data] ….

IMPLEMENTATION NOTE: The mutex must work recursive! That means, that a multiple call out of one task must not block the execution! For each SysMutexEnter() call, a corresponding SysMutexLeave() must be used!

RTS_HANDLE SysMutexCreate (RTS_RESULT *pResult)¶

Create a new mutex object.

Parameters

[out]

pResult

Pointer to error code: One of:

  • ERR_OK: The mutex was created.

  • ERR_NOMEMORY: The needed resources couldn’t be allocated.

Returns:

Handle to the mutex

RTS_RESULT SysMutexDelete (RTS_HANDLE hMutex)¶

Delete a mutex object

Parameters

[in]

hMutex

Handle to the mutex that is provided from SysMutexCreate()

Returns:

One of: ERR_OK: The mutex was deleted successful. ERR_INVALID_HANDLE: The given mutex isn’t a valid mutex handle. ERR_FAILED: An internal error occurred.

RTS_RESULT SysMutexTryEnter (RTS_HANDLE hMutex)¶

Try to enter the mutex. If mutex is available, the function entered the mutex and returned ERR_OK. If mutex is not available, ERR_FAILED is returned.

Parameters

[in]

hMutex

Handle to the mutex that is provided from SysMutexCreate()

Returns:

One of: ERR_OK: The mutex was entered. ERR_INVALID_HANDLE: The given mutex isn’t a valid mutex handle. ERR_FAILED: An internal error occurred. Mutex was no entered.

RTS_RESULT SysMutexEnter (RTS_HANDLE hMutex)¶

Enter the given mutex. If the mutex is still entered by another task1, the actual task2 will be blocked, until the task1 has called SysMutexLeave() .

IMPLEMENTATION NOTE: The mutex must work recursive! That means, that a multiple call out of one task must not block the execution! For each SysMutexEnter() call, a corresponding SysMutexLeave() must be used!

Parameters

[in]

hSem

Handle to the mutex that is provided from SysMutexCreate()

Returns:

One of:

  • ERR_OK: The mutex was entered successful.

  • ERR_INVALID_HANDLE: The given mutex isn’t a valid mutex handle.

  • ERR_FAILED: An internal error occurred.

RTS_RESULT SysMutexLeave (RTS_HANDLE hMutex)¶

Leave the given mutex.

Parameters

[in]

hMutex

Handle to the mutex that is provided from SysMutexCreate()

Returns:

One of:

  • ERR_OK: The mutex was leafed successful.

  • ERR_INVALID_HANDLE: The given mutex isn’t a valid mutex handle.

  • ERR_FAILED: An internal error occurred.
