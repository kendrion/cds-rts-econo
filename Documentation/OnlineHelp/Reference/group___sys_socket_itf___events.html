#define EVT_SysSocket_BeforeSetIpAndMask MAKE_EVENTID ( EVTCLASS_INFO , 1)¶

Event is sent in function SysSockSetIpAddressAndNetMask() before the IP address and network mask is set in the SysSocket OS layer. The following results are provided to the event: ERR_OPERATION_DENIED: Adapter is not white listed to allow address changes. ERR_OK: Adapter is in the white lists for address changes. The result codes ERR_OPERATION_DENIED and ERR_OK can be changed to the opposite to overwrite the adapter list settings. Furthermore it is possible to implement the functionality to set the IP address and network mask completely in this event. In this case ERR_NOTHING_TO_DO have to be returned to signal this to the internal implementation. Then the internal implementation is skipped and SysSockSetIpAddressAndNetMask() will return ERR_OK. The event should only be used by OEMs an not by the application.

#define EVT_SysSocket_AfterSetIpAndMask MAKE_EVENTID ( EVTCLASS_INFO , 2)¶

Event is sent in function SysSockSetIpAddressAndNetMask() after the IP address and network mask was set in the SysSocket OS layer. The event is called in any case regardless, if the operation was successful or not. Typically this is used to store the new IP configuration for example in a system boot up file to keep it also after a reboot. The event should only be used by OEMs an not by the application.

#define EVT_SysSocket_GetAdditionalAdapterInfo MAKE_EVENTID ( EVTCLASS_INFO , 5)¶

Event is sent in function UpdateNetworkAdapterInfo() on startup of the runtime for every InetV4 network adapter. Typically this is used to add the information about how the adapter was configured (static/dhcp) and the DHCP Server Address. Linux Only! The event should only be used by OEMs an not by the application.
