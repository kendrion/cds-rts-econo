Macro Definition Documentation¶

#define CMPSCHEDULE_IECTASKS_HIGH_PRIO -1¶

This is a soft barrier and is not mandatory for the scheduler. But the Timer Scheduler is using this to place the task stacks of tasks with a higher priority (<= CMPSCHEDULE_IECTASKS_HIGH_PRIO) into a different memory area than the others (e.g. internal RAM).

The physical location of the stacks can be influenced with the two macros:

  • CMPSCHEDULE_IECTASK_STACK_STD_ATTRIBUTE

  • CMPSCHEDULE_IECTASK_STACK_HIGH_ATTRIBUTE

#define NUM_OF_HIGH_IEC_TASKS 0¶

Number of high priority tasks, that are allowed below CMPSCHEDULE_IECTASKS_HIGH_PRIO.

#define CMPSCHEDULE_IECTASKS_STACK_STD_ATTRIBUTE¶

Attribute to place the stack of the IEC tasks in a different location.

#define CMPSCHEDULE_IECTASKS_STACK_HIGH_ATTRIBUTE¶

Attribute to place the stack of high priority tasks in a different location (e.g. internal RAM).

#define RTS_CPU_GET_STACK_BASE ((RTS_UINTPTR *)(((RTS_UINTPTR)pStack + stackSize) & ~( SYSCPU_STACK_ALIGNMENT - 1)))¶

Calculate the (aligned) stack base within a buffer. The macro is called when a buffer shall be used as a new stack. Ensure input params use the types as specified. Note: the stack will grow downwards, i.e. from bottom of stack (returned by this macro) to the top of stack (provided by pStack)

Parameters

[in]

pStack

(RTS_UINTPTR) Pointer to the start of buffer containing the stack

[in]

stackSize

(RTS_SIZE) Size of buffer in bytes

Returns:

(RTS_UINTPTR) Pointer to bottom of stack.
