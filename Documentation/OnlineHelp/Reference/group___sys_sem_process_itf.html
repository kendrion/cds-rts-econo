Interface of SysSemProcess.

Detailed Description¶

The SysSemProcess interface is projected to handle synchronization objects for processes. The synchronization objects are called semaphores to synchronize concurrent access to single system global resources.

For example:

Process1: … SysSemProcessEnter(hSem); [Here you can accessed the protected global resource] SysSemProcessLeave(hSem); [After SysSemProcessLeave() you must not access the protected global resource] ….

RTS_RESULT SysSemProcessEnter (RTS_HANDLE hSem, unsigned long ulTimeoutMs)¶

Enter the given semaphore.

Parameters

[in]

hSem

Handle to the semaphore

[in]

ulTimeoutMs

Timeout in milliseconds to wait to enter the semaphore or SYSSEMPROCESS_WAIT_INFINITE for an infinite wait.

Returns:

ERR_OK or ERR_TIMEOUT if timeout expires before enter the semaphore
