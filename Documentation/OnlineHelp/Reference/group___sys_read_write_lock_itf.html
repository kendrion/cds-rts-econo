5.2.144.6. Detailed Description¶

Interface of SysReadWriteLock.

This interface provides a synchronization technique for multiple (shared) reading and single (exclusive) writing access control to a critical resource

5.2.144.8.3. typedef struct tagsysrwlreadlock_struct sysrwlreadlock_struct¶

This function to enter a read lock(synchronization object). NOTE: Several readers can enter the lock simultaneously! But they are synchronized against an entered write lock! RETURN: Returns the runtime system error code (see CmpErrors.library)

5.2.144.8.4. typedef struct tagsysrwlreadlocktry_struct sysrwlreadlocktry_struct¶

This function tries enter a read lock(synchronization object). If a write lock is still active, the functions returns an error. The function succeeded as well, if another read lock is still active. NOTE: Several readers can enter the lock simultaneously! But they are synchronized against an entered write lock! RETURN: Returns the runtime system error code (see CmpErrors.library)

5.2.144.9.11. RTS_HANDLE SysRWLCreate (RTS_RESULT *pResult)¶

This function creates a ReadWrite locking synchronization object.

Parameters

[in] pResult Pointer to a variable to return an errorcode (optional).

Returns:

Returns a handle to the synchronization object or RTS_INVALID_HANDLE in case of failure.

Return values

ERR_OK A valid handle is returned.
ERR_NOMEMORY Not enough memory available.

5.2.144.9.12. RTS_RESULT SysRWLDelete (RTS_HANDLE hRWLock)¶

This function deletes a prior created ReadWrite locking synchronization object.

Parameters

[in] hRWLock Handle to the synchronization object

Returns:

Returns the runtime system error code.

Return values

ERR_OK Success
ERR_INVALID_HANDLE The given handle is invalid

5.2.144.9.15. RTS_RESULT SysRWLReadLockTry (RTS_HANDLE hRWLock)¶

This function tries to enter a read lock (synchronization object).

If a write lock is still active, the functions returns ERR_FAILED. On the other hand the function succeeds, even though another read lock is still active. NOTE: Several readers can enter the lock simultaneously! But they are synchronized against an entered write lock!

Parameters

[in] hRWLock Handle to the synchronization object

Returns:

Returns the runtime system error code.

Return values

ERR_OK Success
ERR_FAILED The lock can’t be entered.
ERR_INVALID_HANDLE The given handle is invalid

5.2.144.9.16. RTS_RESULT SysRWLWriteLockTry (RTS_HANDLE hRWLock)¶

This function tries to enter a write lock (synchronization object).

If a write lock is still active, the functions returns ERR_FAILED. NOTE: Several readers can enter the lock simultaneously! But they are synchronized against an entered write lock!

Parameters

[in] hRWLock Handle to the synchronization object

Returns:

Returns the runtime system error code.

Return values

ERR_OK Success
ERR_FAILED The lock can’t be entered.
ERR_INVALID_HANDLE The given handle is invalid
