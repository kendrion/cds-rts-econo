Macro Definition Documentation¶

#define SYSMCKEY_INT_BINDPROCESS “BindProcess.CoreID”¶

Setting to bind the runtime process to the specified core (0 is the first core). If the value is -1, the process is unbound!

#define SYSMCVALUE_INT_BINDPROCESS -1¶

#define SYSMCKEY_INT_CPULOADTRACE “CpuLoadTrace.Enable”¶

Setting to enable/disable the CPU load system trace feature. With this feature you can upload the system trace “CpuCoreLoad” in CODESYS and you can see the CPU load in a trace window!

#define SYSMCVALUE_INT_CPULOADTRACE 1¶

#define SYSMCKEY_INT_PLCLOADTRACE “PlcLoadTrace.Enable”¶

Setting to enable/disable the PLC load system trace feature. With this feature you can upload the system trace “PlcLoad” in CODESYS and you can see the CPU load in a trace window!

#define SYSMCVALUE_INT_PLCLOADTRACE 1¶

#define SYSMCKEY_STRING_IECCORESET “IecCoreSet”¶

Setting to configure which cores should be used by the IEC code. For notation see: SYSMCSUBKEY_TASKGROUP_CORESET. If not configured the default depends on the available license: 1. No license available: This results in a single core license. Core 0 will be used by default. 2. License available: This will result in a {0..Licensevalue-1} core set. The core set can be overwritten using this setting. Pay attention to configure only the licensed number of cores.

#define SYSMCBASEKEY_TASKGROUP “TaskGroup”¶

A base setting to configure task groups. The following subsettings are available: Name, Option and CoreSet. The numbering starts with 1. Example to bind all system tasks to core 0: TaskGroup.1.Name=System TaskGroup.1.Option=1 TaskGroup.1.CoreSet={0}

#define SYSMCSUBKEY_TASKGROUP_NAME “Name”¶

Setting to name a task group. For predefined task group names see ‘Task group names’. Example: TaskGroup.2.Name=IecTask

#define SYSMCSUBKEY_TASKGROUP_OPTION “Option”¶

Setting to configure the options of a task group. See ‘Task group options’. Example: TaskGroup.2.Option=2

#define SYSMCVALUE_INT_TASKGROUP_OPTION_DEFAULT SYSMC_TASKGROUPOPTION_MULTICORE_FREEFLOATING¶

#define SYSMCSUBKEY_TASKGROUP_CORESET “CoreSet”¶

Setting to configure a set of cores for a task group. There is a placeholder for the highest core id named MAXCOREID to configure independently of the current num of cores. To build more comlex expressions a range and a subtraction operator are provided. Example: TaskGroup.2.CoreSet={0..MAXCOREID-1} TaskGroup.3.CoreSet={0,1,2}
