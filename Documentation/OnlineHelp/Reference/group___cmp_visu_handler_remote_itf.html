typedef RTS_BOOL( * PFVishRemoteProvideCredentials) (RTS_HANDLE hCallerData, char *pszUserBuffer, RTS_SIZE userBufferSize, char *pszPasswordBuffer, RTS_SIZE passwordBufferSize)¶

This function will be called if it is necessary to login to the PLC or a previous login attempt failed.

Parameters

[in]

hCallerData

An arbitrary object that will be forwarded to the callback functions

[in]

pszUserBuffer

A buffer for the user name. Will not be NULL.

[in]

userBufferSize

The size of pszUserBuffer in bytes.

[in]

pszPasswordBuffer

A buffer for the password. Will not be NULL.

[in]

passwordBufferSize

The size of pszPasswordBuffer in bytes.

Returns:

Return FALSE to use the credentials. Return TRUE to cancel the surrounding operation.

typedef VishRemoteVertifyCertResult( * PFVishRemoteVerifyCertificate) (RTS_HANDLE hCallerData, RTS_HANDLE hCert, RTS_RESULT preVerifyResult)¶

This function will be called during connection setup in case encrypted communication is used. The intention of the call is to verify the certificate that was returned by the connected PLC.

Parameters

[in]

hCallerData

The arbitrary object that was passed as hCallerData in VishRemoteCertRegisterVerifyCallback

[in]

hCert

The handle to the X509 certificate that has to be verified

[in]

preVerifyResult

The result of the verification done by the system (like checking for known certificates, expired certificates). Depending on this first verification step and the content of the certificate the implementation of this function can decide how to proceed.

Returns:

One of the values from VishRemoteVertifyCertResult that signals how to proceed with the certificate and the according communication setup

Function Documentation¶

RTS_RESULT VishRemoteConnect (RTS_BOOL useTempSettings)¶

This function can be called to start the remote target-visualization in a delayed way. Typically this is done when the event EVT_CmpVisuHandlerRemote_CheckAutoStartup is handled to delay the initial startup.

Parameters

[in]

useTempSettings

The connection relevant settings like address or name will be taken from settings prefixed with the prefix “TEMP.” instead of the original name if this value is TRUE. The idea is that temporary settings can be realized without overwriting possibly existing persistent settings.

This call will fail if the remote target-visualization is already running.

Returns:

error code

RTS_RESULT VishRemoteReadApplications (RTS_HANDLE hCallerData, PFVishRemoteProvideCredentials pfProvideCredentials, PFVishRemoteNotifyApplicationName pfAppCallback, RTS_BOOL bReadTempSettings)¶

This function reads the available applications from the connected PLC and returns them using the configured callback functions.

Parameters

[in]

hCallerData

An arbitrary object that will be forwarded to the callback functions

[in]

pfProvideCredentials

The callback function that will be called in case a login is required. Must not be NULL.

[in]

pfAppCallback

The callback function that will be called for each application on the connected PLC. Must not be NULL.

[in]

bReadTempSettings

If TRUE then temporary settings (beginning with “Temp.”) are read from the configuration file are read, otherwise the “normal” settings

The callbacks will be called synchronously

Returns:

error code

RTS_RESULT VishRemoteCertFormatTextual (RTS_HANDLE hCert, char *pszBuffer, RTS_SIZE bufferSize)¶

This utility function can be used to format the contents of an X509 certificate so that the most relevant information can be presented to the user.

Parameters

[in]

hCert

The handle to the X509 certificate that should be formatted in a textual way.

[in]

pszBuffer

The buffer that will be used for formatting the certificate information. Must not be NULL.

[in]

bufferSize

The size of the buffer (bufferSize

) that will be used for formatting the certificate information.

The passed buffer should be sufficiently large (maybe ~1000 bytes) so that all relevant information of the certificate can be included.

Returns:

error code

RTS_RESULT VishRemoteCertRegisterVerifyCallback (RTS_HANDLE hCallerData, PFVishRemoteVerifyCertificate pfCallback)¶

This function registers the given callback so that unknown certificates can be verified by the user if encrypted communication is used.

Parameters

[in]

hCallerData

An arbitrary object that will be forwarded to the callback function

[in]

pfCallback

The callback function that will be called in case a certificate has to be verified. Must not be NULL.

The callbacks will be called synchronously from an arbitrary thread.

Returns:

error code

RTS_RESULT VishRemoteCertExport (RTS_HANDLE hCert, char *pszPathBuffer, RTS_SIZE bufferSize)¶
