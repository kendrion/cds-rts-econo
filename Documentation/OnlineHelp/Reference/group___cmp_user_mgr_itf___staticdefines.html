Macro Definition Documentation¶

#define USERMGR_NUM_OF_STATIC_CREDENTIAL_CONTEXTS 4¶

Number of predefined static login credential contexts used for authentication

#define USERMGR_NUM_OF_STATIC_SESSIONS 8¶

Number of predefined static session entries

#define USERMGR_MAX_PASSWORD_LEN 60¶

Max. length of the passwords. This value is used as buffer size and includes the terminating NUL.

#define USERMGR_MAX_USERNAME_LEN 60¶

Max. length of the user names. This value is used as buffer size and includes the terminating NUL.

#define USERMGR_MAX_GROUPNAME_LEN 60¶

Max. length of the user group names. This value is used as buffer size and includes the terminating NUL.

#define SRV_READ_USER_MGMT 0x01¶

#define SRV_WRITE_USER_MGMT 0x02¶

#define USERDB_OBJECT_USERMGMT “Device.UserManagement”¶

Predefined objects in the runtime

#define USERMGR_ASYMMETRIC_KEY_UPDATE_INTERVAL (24*60*60*1000) /* 24h */¶

Defines the update interval in ms for the key pair used by the asymmetric authentication algorithm. Whenever this time elapses since the startup of the device or the last key generation, the CmpUserMgr generates a new pair of keys in the background.
