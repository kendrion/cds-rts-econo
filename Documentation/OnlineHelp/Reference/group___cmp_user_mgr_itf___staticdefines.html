#define USERMGR_NUM_OF_STATIC_SESSIONS 8¶

Number of predefined static session entries

#define SRV_READ_USER_MGMT 0x01¶

#define USERDB_OBJECT_USERMGMT “Device.UserManagement”¶

Predefined objects in the runtime

#define USERMGR_NUM_OF_LOGIN_RETRIES 3¶

Number of unsuccessful login retries to limit failed user authentication In case USERMGR_NUM_OF_LOGIN_RETRIES is set to 0 this security feature is disabled.

#define USERMGR_USERLOCK_TIME 60¶

Timeout after USERMGR_NUM_OF_LOGIN_RETRIES unsuccessful login tries in seconds, i.e. the user is temporarily excluded The maximal timeout value for user exclusion is 4233600 seconds which is equivalent to 49 days. But this is not persistent, i.e. a restart of the runtime resets the exclusion.
