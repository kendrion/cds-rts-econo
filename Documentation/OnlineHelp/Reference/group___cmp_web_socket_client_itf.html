Interface of CmpWebSocketClient.

Detailed Description¶

This component contains an implementation of the WebSocket protocol on the client side. See RFC-6455 ( https://tools.ietf.org/html/rfc6455) for details.

The following features are supported:

  • Connect to WebSocket server using plaintext connection. (ws:// schema)

  • Connect to WebSocket server using TLS connection. (wss:// schema)

  • Send binary and text frames.

  • Do a fragmented transfer. Fragmentation has to be done by using implementation.

  • Receive binary and text frames.

  • Receive a fragmented transfer. Defragmentation has to be done by using implementation;

  • Ping / Pong is handled by the implementation. The using implementation is notified if a Ping or Pong was received.

The configuration of the TLS connection has to be done by the using implementation. A TLS context must be configured to the needs of the application. This TLS context is passed to the WebSocket implementation. It is used to create a secure socket. If needed a callback for the server certificate validation can be added to this TLS context. The TLS context must use a TLS client method.

The Implementation will parse the given URL and get the host (or IP-address), port (if set) and path (if set) from that URL. If no port is given default ports will be used. (80 for ws:// schema, 443 for wss:// schema).

typedef void( * PFDATARECEIVEDCALLBACK) (RTS_HANDLE hClient, void *pUserData, RTS_UI8 *pData, RTS_SIZE dataSize, RTS_BOOL bBinaryMode, RTS_BOOL bFinalFragmet)¶

Callback function if a complete frame is received from the WebSocket. A single frame is completely defragmented by the WebSocket implementation. A fragmented transfer where multiple frames are used must be defragmented by the caller.

Parameters

[in]

hClient

Handle to the client that received the frame.

[in]

pUserData

Pointer to user defined data.

[in]

pData

Pointer to the received data.

[in]

dataSize

Size of available data.

[in]

bBinaryMode

TRUE if a binary frame is received. FALSE if a text frame is received.

[in]

bFinalFragmet

TRUE if this is the final fragment. This can happen for the first frame. If FALSE this is part of a fragmented transfer.

RTS_HANDLE WsClientCreate (RTS_SIZE bufferSize, char *pszUrl, char *pszProto, RTS_HANDLE hTlsContext, void *pUserData, PFDATARECEIVEDCALLBACK pfDataReceivedCallback, PFSTATECHANGECLALLBACK pfStateChangeCallback, RTS_RESULT *pResult)¶

Create a new WebSocket client.

Parameters

[in]

bufferSize

Set this to the maximum data size size needed.

[in]

pszUrl

URL to connect to. The official scheme for WebSockets ws://host[:port][/path] (plain text) or wss://[:port][/path] (TLS connection) is expected here. The parts are used in that way: host (Mandatory): Used for HOST in the HTTP GET request. Used to resolve the Server. You can use a host name or IP-address. If a host name is used and scheme is WSS a SNI will be added to the TLS handshake. port (Optional): Port to connect to. path (Optional): Path is used in the HTTP GET request. If not available a ‘/’ will be added to the HTTP GET request.

[in]

pszProto

Protocol used inside the websocket handshake header. May be NULL.

[in]

hTlsContext

The TLC context to use if a wss:// scheme is given.

[in]

pUserData

Pointer to a user defined value. This is passed to the two callbacks.

[in]

pfDataReceivedCallback

Callback function used if a data frame is received.

[in]

pfStateChangeCallback

Callback function used to send state changes.

[out]

pResult

Error Code:

  • ERR_OK: WebSocket client created successfully.

  • ERR_PARAMETER: One of the parameters is invalid.

  • ERR_INVALID_URL: The given URL is invalid.

  • ERR_INVALID_HANDLE: A wss:// scheme is given, but hTlsContext is invalid.

Returns:

Handle to the WebSocket. RTS_INVALID_HANDLE in case of an error.

RTS_RESULT WsClientConnect (RTS_HANDLE hClient)¶

Connects the WebSocket to the WebSocket server.

Parameters

[in]

hClient

Handle to the WebSocket

Returns:

Error Code:

  • ERR_INVALID_HANDLE: The given hClient is unknown.

  • ERR_SOCK_HOSTNOTFOUND: The host of the URL could not be found.

  • ERR_FAILED: Some internal error occurred.

  • ERR_PENDING: Connection pending. Will be finished within WsClientProcess. Wait for state callback with WsClientState_Connected.

RTS_RESULT WsClientDisconnect (RTS_HANDLE hClient, WebSocketStatus statusCode)¶

Disconnects the WebSocket. Sends a FIN frame to the server. Use WsClientConnect to reestablish the connection the same server.

Parameters

[in]

hClient

Handle to the WebSocket.

[in]

statusCode

Status code why the connection is beeing closed.

Returns:

Error Code:

  • ERR_INVALID_HANDLE: The given hClient is unknown.

  • ERR_OK: The connection is closed.

RTS_RESULT WsClientSend (RTS_HANDLE hClient, RTS_UI8 *pSendData, RTS_SIZE dataSize, RTS_BOOL bBinaryMode, RTS_BOOL bFinalFragmet)¶

Send a data frame to the server.

Parameters

[in]

hClient

Handle to the WebSocket.

[in]

pSendData

Pointer to the data to send.

[in]

dataSize

Size of the data to send.

[in]

bBinaryMode

If TRUE, a binary frame will be sent. If FALSE, a text frame will be sent.

[in]

bFinalFragmet

Use this to signal that this is the final frame. If not set the subsequent calls will send send continuation frames until bFinalFragmet is set to TRUE.

Returns:

Error Code:

  • ERR_OK: Frame is sent.

  • ERR_BUFFERSIZE: Too much data to send.

  • ERR_PENDING: Parts where sent. Sending is finished in WsClientProcess.

  • ERR_CALL_AGAIN: No data sent. Call again.

hClient

Handle to the WebSocket

[in]

ui32Timeout

Timeout used for SysSockSelect.

Returns:

Error Code:

  • ERR_INVALID_HANDLE: The hClient is unknown.

RTS_RESULT WsClientGetHttpHeader (RTS_HANDLE hClient, RTS_UI8 **ppHeader)¶

Function to retrieve a pointer to the header of the initial HTTP response. Can only be called in the context of the callback PFSTATECHANGECLALLBACK during the state WsClientState_InConnect_InitialResponseReceived.

Parameters

[in]

hClient

Handle to the WebSocket

[out]

ppHeader

Pointer to Pointer to the retrieved HTTP header

Returns:

Error Code:

  • ERR_INVALID_HANDLE: The hClient is unknown.

  • ERR_PARAMETER: ppHeader is NULL.

  • ERR_INVALID_STATE: The client is not in the state WsClientState_InConnect_InitialResponseReceived.

  • ERR_OK: The HTTP header has been successfully retrieved.
