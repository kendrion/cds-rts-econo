5.2.136.5. Detailed Description¶

Interface of SysModule.

The SysModule interface is projected to handle component modules. This interface is only available on systems, that supports dynamically loading and unloading modules.

5.2.136.8.3. RTS_HANDLE SysModuleLoad (char *pszModuleName, RTS_RESULT *pResult)¶

Load a specified module

Parameters

[in] pszModuleName Pointer to the module name. NOTE: Module names MUST NOT include operating system specific endings (e.g. .dll, .o, etc.)
[out] pResult Pointer to error code

Returns:

Handle to the loaded module or RTS_INVALID_HANDLE if failed.

5.2.136.8.4. RTS_HANDLE SysModuleLoad2 (char *pszModuleName, char *pszModuleFileName, RTS_RESULT *pResult)¶

Load a specified module

Parameters

[in] pszModuleName Pointer to the module name. NOTE: Module names must _NOT_ include operating system specific endings (e.g. .dll, .o, etc.)
[in] pszModuleFileName Pointer to the module file name. NOTE: Module names _must_ include operating system specific endings (e.g. .dll, .o, etc.) and optional the path of the file!
[out] pResult Pointer to error code

Return values

ERR_OK Module could be loaded
ERR_FAILED Failed to load module
ERR_PARAMETER Invalid parameter (pszModuleName = NULL/empty or pszModuleFileName = NULL/empty)

Returns:

Handle to the loaded module or RTS_INVALID_HANDLE if failed
