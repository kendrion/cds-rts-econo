7.3.17.6. generate_as_separate_applicationÂ¶

Name: symbolconfiguration\generate_as_separate_application

Required Runtime Components: CmpIecVarAccess

Default Value: True

Recommended Value: False

XML Type: boolean

dotNet Type: System.Boolean

Example:

<ts:section name="symbolconfiguration">
      ...
    <ts:setting name="generate_as_separate_application" type="boolean" access="visible">
        <ts:value>0</ts:value>
    </ts:setting>
      ...
</ts:section>

Documentation:

Boolean:
			0: Symbolconfiguration is generated in father application [recommended setting for new devices]
			1: Symbolconfiguration is generated as a separate child application [Default]
			(This setting is only valid if 'symbolconfiguration\\transfer_xml_only' is false.

PlugIn:

SymbolConfigObject.plugin (3.5.17.0)
