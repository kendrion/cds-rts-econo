6.19.1. allow_calls_through_opcuaÂ¶

Name: symbolconfiguration\allow_calls_through_opcua

Runtime Version: 3.5.11.0

Compiler Version: 3.5.11.0

Required Runtime Components: CmpIecVarAccess

Default Value: Forbidden

Recommended Value: Forbidden

XML Type: string

dotNet Type: System.String

Example:

<ts:section name="symbolconfiguration">
      ...
    <ts:setting name="allow_calls_through_opcua" type="string" access="visible">
        <ts:value>Forbidden</ts:value>
    </ts:setting>
      ...
</ts:section>

Documentation:

Setting whether the runtime supports calls into IEC code via IecVarAccess (e. G. for extended UPC UA support).
                This setting will be ignored if 'symbolconfiguration\include_opcua_information' is set to 'Forbidden', or 'symbolconfiguration\generate_as_separate_application' is not set to 'false'. 
                - 'Forced': The feature is selected, and the user cannot edit it. (This value is not recommended, as this feature implies some compile-time overhead.)
                - 'Enabled': The feature is selected, but the user can change it.
                - 'Allowed': The feature is disabled, but the user can change it. (This is the recommended value if the runtime actually supports calls, e. G. through an OPC UA Server or other extension.)
                - 'Forbidden': The feature is disabled, and the user cannot enable it. (This is the default value if the setting is not present, and also hides or disables the corresponding settings in the wizard and the symbol config editor.)

PlugIn:

SymbolConfigObject.plugin (3.5.15.0)
