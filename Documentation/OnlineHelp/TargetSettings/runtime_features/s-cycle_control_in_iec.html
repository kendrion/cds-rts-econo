7.3.15.17. cycle_control_in_iecÂ¶

Name: runtime_features\cycle_control_in_iec

Default Value: False

Recommended Value: False

XML Type: boolean

dotNet Type: System.Boolean

Example:

<ts:section name="runtime_features">
      ...
    <ts:setting name="cycle_control_in_iec" type="boolean" access="visible">
        <ts:value>0</ts:value>
    </ts:setting>
      ...
</ts:section>

Documentation:

1 = Task FB is always called, irrespective of the application status.
The task FB uses this switch to check the status of the task and no
longer calls the task POUs in stop state. 0 = CmpApp in the runtime
system must evaluate the status of an application and call the task FB
accordingly.

PlugIns:

DeviceObject.plugin (3.5.17.0)
LanguageModelManager.plugin (3.5.17.0)
TaskConfigObject.plugin (3.5.17.0)
