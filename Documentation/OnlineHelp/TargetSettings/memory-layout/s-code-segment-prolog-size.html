7.3.11.16. code-segment-size

7.3.11.15. code-segment-prolog-sizeÂ¶

Name: memory-layout\code-segment-prolog-size

Default Value: 0

Recommended Value: 0

XML Type: integer

dotNet Type: System.Int32

Example:

<ts:section name="memory-layout">
      ...
    <ts:setting name="code-segment-prolog-size" type="integer" access="visible">
        <ts:value>0</ts:value>
    </ts:setting>
      ...
</ts:section>

Documentation:

If this  value is != 0, the specified size at the beginning of each code 
area is not used for allocation of IEC- Code. Thus the runtime 
system is free to add specific code in this place. 
At the moment this setting is only used for ARM- Targets, in this 
case the prolog contains a jump to the breakpoint handling routine 
in the runtime system. The prolog must have a size of 12 bytes, 
if it is smaller code of the IEC app is overwritten.
The code patched for Breakpoints is a jump to offset 0 in the area. 
Therefore we can avoid Exception handling 
for handling breakpoints.
