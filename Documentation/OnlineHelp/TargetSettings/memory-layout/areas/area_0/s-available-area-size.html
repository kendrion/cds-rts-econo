Name: memory-layout\areas\area_{0}\available-area-size

Default Value: -1

Recommended Value: -1

XML Type: integer

dotNet Type: System.Int32

Example:

<ts:section name="memory-layout">
      ...
    <ts:section name="areas">
          ...
        <ts:section name="area_{0}">
              ...
            <ts:setting name="available-area-size" type="integer" access="visible">
                <ts:value>-1</ts:value>
            </ts:setting>
              ...
        </ts:section>
          ...
    </ts:section>
      ...
</ts:section>

Documentation:

If > 0: this is the maximal size that is available for download, this is the total amount of data, that will be allowed for download or online change. Any additional size is only for fragmentation.
This is a new option for areas. The area should be of a fixed size. E.g: 
<ts:section name="area_1"> 
<!-- Global code memory --> 
<ts:setting name="flags" type="integer" access="visible"> 
<ts:value>0x0040</ts:value> 
</ts:setting> 
<ts:setting name="minimal-area-size" type="integer" access="visible"> 
<ts:value>0x20000</ts:value> 
</ts:setting> 
<ts:setting name="maximal-area-size" type="integer" access="visible"> 
<ts:value>0x20000</ts:value> 
</ts:setting> 
<ts:setting name="available-area-size" type="integer" access="visible"> 
<ts:value>0x10000</ts:value> 
</ts:setting> 
<ts:setting name="allocation-plus-in-percent" type="integer" access="visible"> 
<ts:value>0</ts:value> 
</ts:setting> 
</ts:section> 

Then the available size will be checked for each download and online change. In the upper example, the code size is not allowed to increase the 0x10000. The additional size is only allowed for online changes. 
If a POU is changed by online change, the code is allocated for the old and the new POU, but the calculated size is only considering the new POU. 

Thus you avoid the situation, that an online change is not possible, but a Download would be possible. 
Note that the code calculation is not 100 % correct. There might be minor differences between the code size calculated at online change, and the code size really used for download.
