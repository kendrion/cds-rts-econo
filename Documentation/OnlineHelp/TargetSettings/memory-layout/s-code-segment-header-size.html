7.3.11.14. code-segment-header-sizeÂ¶

Name: memory-layout\code-segment-header-size

Default Value: 0

Recommended Value: 0

XML Type: integer

dotNet Type: System.Int32

Example:

<ts:section name="memory-layout">
      ...
    <ts:setting name="code-segment-header-size" type="integer" access="visible">
        <ts:value>0</ts:value>
    </ts:setting>
      ...
</ts:section>

Documentation:


With the setting -> compact-download, a header will be generated in the code segment. With this setting the size of this header is fixed.
Some codegenerators need a fixed header size for each memory segment containing code. (E.g. this is necessary for ARM-targets).

PlugIns:

LanguageModelManager.plugin (3.5.17.0)
OnlineManager.plugin (3.5.17.0)
