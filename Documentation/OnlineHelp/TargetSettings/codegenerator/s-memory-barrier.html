7.3.2.35. memory-barrierÂ¶

Name: codegenerator\memory-barrier

Default Value: False

Recommended Value: False

XML Type: boolean

dotNet Type: System.Boolean

Example:

<ts:section name="codegenerator">
      ...
    <ts:setting name="memory-barrier" type="boolean" access="visible">
        <ts:value>0</ts:value>
    </ts:setting>
      ...
</ts:section>

Documentation:

Code is generated for operator __MemoryBarrier. Implemented for x86, x86-64, ARM, ARM-Thumb2. For PowerPC
                with compilerversion >= 3.5.16.0.
                For all other codegenerators the operator is replaced by an empty instruction without a breakpoint position.
                For x86 a 'mfence' instruction is generated. For ARM a DMB (data memory barrier) instruction. 
                The target setting is necessary because not all versions of the cpu supports these instructions.                
                In general a memory barrier may be necessary in parallel processing because of out-of-order execution of instructions.

PlugIn:

LanguageModelManager.plugin (3.5.17.0)
