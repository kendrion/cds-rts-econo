6.2.13. Breakpoint SizeÂ¶

Name: codegenerator\Breakpoint Size

Default Value: 8

Recommended Value: 4

XML Type: integer

dotNet Type: System.Int32

Example:

<ts:section name="codegenerator">
      ...
    <ts:setting name="Breakpoint Size" type="integer" access="visible">
        <ts:value>4</ts:value>
    </ts:setting>
      ...
</ts:section>

Documentation:

Very special setting to define the lenght of a breakpoint instruction in bytes.
                The default depends on the selected codegenerator and normally the value should not be changed.
                For x86 code generation it must be 5.

PlugIns:

Code386.plugin (3.5.15.0)
CodeMIPS.plugin (3.5.15.0)
CodeSH.plugin (3.5.15.0)
