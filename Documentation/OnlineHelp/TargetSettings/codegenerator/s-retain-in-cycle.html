7.3.2.41. retain-in-cycleÂ¶

Name: codegenerator\retain-in-cycle

Default Value: False

Recommended Value: False

XML Type: boolean

dotNet Type: System.Boolean

Example:

<ts:section name="codegenerator">
      ...
    <ts:setting name="retain-in-cycle" type="boolean" access="visible">
        <ts:value>0</ts:value>
    </ts:setting>
      ...
</ts:section>

Documentation:

If enabled, all retain variables will be allocated in normal memory, and copied at the end of each cycle to the retain memory.
                With this option, a single retain variable in a function block instance will not cause the whole function block instance
                being allocated to retain memory.
                If the value for this setting is 'true' and compiler version is >= 3.2.2.20, then
                all retain parameters will be handled in the following way:
                - All retain variables (not only that for function blocks) are located in the standard data memory
                - Additional code is generated for all retain variable (especially for all
                  retains in FB-instances) to copy all retains at the end of a cycle to the retain area.
                - After booting with a project the retain variables are initialized with the data in retain memory.
                As a consequence, it is now also possible to declare variables on direct output addresses as RETAIN variables.
                Note: This behavior may have effects on code size and performance.

PlugIn:

LanguageModelManager.plugin (3.5.17.0)
