6.2.59. maximum_num_applicationsÂ¶

Name: codegenerator\maximum_num_applications

Default Value: 2147483647

Recommended Value: 2147483647

XML Type: integer

dotNet Type: System.Int32

Example:

<ts:section name="codegenerator">
      ...
    <ts:setting name="maximum_num_applications" type="integer" access="visible">
        <ts:value>2147483647</ts:value>
    </ts:setting>
      ...
</ts:section>

Documentation:

Maximum number of applications, which can be used in the project without
                getting a warning at compile time.Also implicitly generated applications will
                be regarded, like e.g. symbol application. You might use this setting to
                control the number of applications for download to the target system. Note
                however that the applications for a certain target may be distributed over
                several projects, so that a download might fail even if there has been no
                warning at compilation of a particular project.

PlugIns:

Compiler.plugin (3.5.15.0)
LanguageModelManager.plugin (3.5.15.0)
