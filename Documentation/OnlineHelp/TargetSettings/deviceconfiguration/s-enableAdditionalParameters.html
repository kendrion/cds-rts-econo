6.6.14. enableAdditionalParametersÂ¶

Name: deviceconfiguration\enableAdditionalParameters

Default Value: False

Recommended Value: False

XML Type: boolean

dotNet Type: System.Boolean

Example:

<ts:section name="deviceconfiguration">
      ...
    <ts:setting name="enableAdditionalParameters" type="boolean" access="visible">
        <ts:value>0</ts:value>
    </ts:setting>
      ...
</ts:section>

Documentation:

Type: boolean; TRUE or 1: 
Additional parameters are automatically created for all devices downloaded to the runtime system. 
Parameter with ID 0x70000010 contains the device name as in the device tree.
Parameter with ID 0x70000011 contains the hashcode of the device identification and is used in the extended online config mode: 
If already an application is on the PLC, the user gets a dialog, where he can choose between connecting to the PLC via ???Parameter mode??? and via ???Application mode???. 
Parameter mode: The project structure on the PLC will be compared with that in the project, and if there is no mismatch, a connection to the PLC will be established. In the generic parameter editor dialog the parameters can be read and written. Applications keep untouched. 
Application mode: The ???online configuration mode??? will be established, i.e. an implicit application will be generated on the device for test purposes. No real application program must be downloaded.
Parameter with the ID 0x70000012 constains the setting UpdateIoInStop for the plc device
FALSE or 0: No additional parameters are created and no ???Parameter mode??? available (like if you choose ???online config mode??? when there is no application on the PLC)

PlugIn:

DeviceObject.plugin (3.5.15.0)
