Detailed Description¶

Example for defining and registering supervisor operations.

This example registers two vital operations to the Supervisor: 1) an operation that cyclically triggers the alive signal in the CH_COMM_CYCLE hook. 2) an operation sets the dead signal after a configured time after the start of the supervision.

Function Documentation¶

static RTS_RESULT RegisterSupervisor (void)¶

Function to register the above operations to the supervisor. USAGE:

  • Must be called in CH_INIT3 out of component HookFunction()!

Returns:

error code

static RTS_RESULT DisableSupervisor (void)¶

Function to disable the above operations at the supervisor. USAGE:

  • Should be called as soon as the runtime starts the shutdown, here in CH_EXIT_COMM out of component HookFunction()!

Returns:

error code

static RTS_RESULT UnregisterSupervisor (void)¶

Function to unregister the above operations from the supervisor. USAGE:

  • Must be called in CH_EXIT3 out of component HookFunction()!

Returns:

error code

static RTS_RESULT CyclicSupervisorOperations (void)¶

Function for the cyclic handling of the registered supervisor operations. USAGE:

  • Must be called in CH_COMM_CYCLE out of component HookFunction()!

Returns:

error code
