^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

Routine handle a debug breakpoint. This routine is typically called from the :ref:`syscpudebughandler() <group___sys_cpu_handling_itf_1gaf578b1b3a5575decb892beb9ce698a90>` method of the SysCpuHandling component. This function handles all debugging features (entering/leaving a breakpoint, set next statement, flowcontrol, executionpoints, etc.)

**Parameters**

+---------+----+------------------------------------------------------------------------------------------------------------------------------+
| [in]    | SP | Actual stack pointer                                                                                                         |
+---------+----+------------------------------------------------------------------------------------------------------------------------------+
| [in]    | BP | Frame pointer or base pointer to the stack, where the first entry function right after the debug code begins its stack usage |
+---------+----+------------------------------------------------------------------------------------------------------------------------------+

**Returns:**

Return address, to which we would like to return right after leaving this function. NOTE: This must be adopted by the caller!
