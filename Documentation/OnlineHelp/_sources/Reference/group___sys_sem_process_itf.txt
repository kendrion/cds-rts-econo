Detailed Description
--------------------

Interface of SysSemProcess.

The SysSemProcess interface is projected to handle synchonization objects for processes. The synchronization objects are called semaphores to synchronize concurrent access to single system global resources.

For example:

Process1: ... SysSemProcessEnter(hSem); [Here you can accessed the protected global resource] SysSemProcessLeave(hSem); [After :ref:`SysSemProcessLeave() <group___sys_sem_process_itf_1ga255ec899cf363d733ee7b88d87f6c0db>` you must not access the protected globale resource] ....
