#define CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_PLCNAME "Communication.PlcNameDest"
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

The name of the plc this remote client should connect to If this mode is used, then a network scan will be used for finding the according device. When an explicit communication address is assigned in CMPVISUHANDLERREMOTE_KEY_VISURUNTIME_ADDRESS, then this setting will be ignored. To be able to use this feature, the component CmpNameServiceClient must be available!

#define CMPVISUHANDLERREMOTE_KEY_VISUCOMM_EXECUTE_APPLOGIN "Communication.ExecuteApplicationLogin"
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

This setting specifies whether an application login is done during connecting to the plc. This login is necessary to check for a currently executing download that might take a longer time to prevent followup issues like missing images. In case this application login is not possible (maybe due to accessrights) and therefore should be omitted at all, this value can be set to 0.

The obfuscated value of the user's password.

As the application must be able to decrypt the password to send it to the connected plc, we can use only a symmetric encryption algorithm. This is merely the prevention of users having a chance of looking at the configuration file to see the password in cleartext. This setting will be created automatically during startup of the remote targetvisualization if CMPVISUHANDLERREMOTE_KEY_VISUCREDENTIALS_PASSWORD
