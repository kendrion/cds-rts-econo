Detailed Description
--------------------

Interface of SysReadWriteLock.

This interface provides a synchronization technique for multiple (shared) reading and single (exclusive) writing access control to a critical resource

RTS_HANDLE SysRWLCreate (RTS_RESULT *pResult)
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

This function creates a ReadWrite locking synchronization object.

**Parameters**

+---------+---------+----------------------------------------------------------+
| [in]    | pResult | Pointer to a variable to return an errorcode (optional). |
+---------+---------+----------------------------------------------------------+

**Returns:**

Returns a handle to the synchronization object or RTS_INVALID_HANDLE in case of failure.

**Return values**

+--------------+------------------------------+
| ERR_OK       | A valid handle is returned.  |
+--------------+------------------------------+
| ERR_NOMEMORY | Not enough memory available. |
+--------------+------------------------------+

RTS_RESULT SysRWLReadLockTry (RTS_HANDLE hRWLock)
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

This function tries to enter a read lock (synchronization object).

If a write lock is still active, the functions returns ERR_FAILED. On the other hand the function succeeds, even though another read lock is still active. NOTE: Several readers can enter the lock simultaneously! But they are synchronized against an entered write lock!

**Parameters**

+---------+---------+--------------------------------------+
| [in]    | hRWLock | Handle to the synchronization object |
+---------+---------+--------------------------------------+

**Returns:**

Returns the runtime system error code.

**Return values**

+--------------------+-----------------------------+
| ERR_OK             | Success                     |
+--------------------+-----------------------------+
| ERR_FAILED         | The lock can't be entered.  |
+--------------------+-----------------------------+
| ERR_INVALID_HANDLE | The given handle is invalid |
+--------------------+-----------------------------+
