================
SysSemProcessItf
================

.. contents::
    :local:
    :depth: 1

Interface of SysSemProcess.

Detailed Description
--------------------

The SysSemProcess interface is projected to handle synchronization objects for processes. The synchronization objects are called semaphores to synchronize concurrent access to single system global resources.

For example:

Process1: ... SysSemProcessEnter(hSem); [Here you can accessed the protected global resource] SysSemProcessLeave(hSem); [After :ref:`SysSemProcessLeave() <group___sys_sem_process_itf_1ga255ec899cf363d733ee7b88d87f6c0db>` you must not access the protected global resource] ....
