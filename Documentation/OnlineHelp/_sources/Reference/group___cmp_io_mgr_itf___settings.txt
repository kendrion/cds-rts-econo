.. _group___cmp_io_mgr_itf___settings_1ga4324170e4c681b618f530229aeaf7940:

#define IOMGRKEY_INT_ENABLE_FULL_CONSISTENCY "EnableFullConsistency"
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

Setting to enable full IO-update consistency resp. disable DRVPROP_CONSISTENCY for all IO-drivers! This setting must be used, if consistency cannot be realized on this target by the IO-driver itself. NOTE: This settings must be enabled on targets with no strict priority scheduling for the IO-update tasks (for MultiCore targets) or on systems with 2 buscycle tasks (for SIL3).
