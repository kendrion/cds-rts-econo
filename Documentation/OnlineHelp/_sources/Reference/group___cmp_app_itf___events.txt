.. _group___cmp_app_itf___events_1gafac87d87339e3acc5942b46c2b1af8cc:

#define EVT_AllocArea :ref:`MAKE_EVENTID <group___cmp_event_mgr_itf___convertmacro_1ga8f45ad6798a634d4e00140c12039007e>` ( :ref:`EVTCLASS_INFO <group___cmp_event_mgr_itf___eventclasses_1ga711863810010aec40693ca7c4643a025>` , 50)
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

This event allows to specify an area address in an OEM runtime component. Overloading or modifying the system component SysMem can lead to problems, and is not needed any more when using this event. For example, to specify an address for a separate M area,

- define a separate M area in your device description with setting area set to Memory = 0x10

- In this event, check this area type with "IsArea(pParam->usType, DA_MEMORY)" and return your memory pointer in pArea. The Event is sent before call to SysMemAllocArea. As parameter, a pointer to :ref:`EVTPARAM_CmpAppArea <struct_e_v_t_p_a_r_a_m___cmp_app_area>` is passed. If structure member pArea is set to a value <> NULL, SysMemAllocArea (or other allocation function) is not called and this value is used for the area. The Event is also sent before call to SysMemFreeArea, in case and pArea was allocated by the first call. In this case, free the memory that was allocated in EVT_AllocArea.

**Parameters**

+---------+-------------+--------------------------------------------------------------------+
| [in]    | pEventParam | :ref:`EVTPARAM_CmpAppArea <struct_e_v_t_p_a_r_a_m___cmp_app_area>` |
