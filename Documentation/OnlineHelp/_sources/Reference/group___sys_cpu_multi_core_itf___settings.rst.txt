.. _group___sys_cpu_multi_core_itf___settings_1gad2228d4d2f57a91a421a200d9054ae1f:

#define SYSMCKEY_STRING_IECCORESET "IecCoreSet"
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

<type>String</type>

Setting to configure which cores should be used by the IEC code. For notation see: SYSMCSUBKEY_TASKGROUP_CORESET. If not configured the default depends on the available license: 1. No license available: This results in a single core license. Core 0 will be used by default. 2. License available: This will result in a {0..Licensevalue-1} core set. The core set can be overwritten using this setting. Pay attention to configure only the licensed number of cores.

.. _group___sys_cpu_multi_core_itf___settings_1gac7dcd80e80a7d39a17af4dfadbf2702e:

#define SYSMCBASEKEY_TASKGROUP "TaskGroup"
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

A base setting to configure task groups. The following sub-settings are available: Name, Option and CoreSet. The numbering starts with 1. Example to bind all system tasks to core 0: TaskGroup.1.Name=System TaskGroup.1.Option=1 TaskGroup.1.CoreSet={0}

.. _group___sys_cpu_multi_core_itf___settings_1ga6b91d533774b3707c77a998d28e11b1f:

#define SYSMCSUBKEY_TASKGROUP_CORESET "CoreSet"
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

<type>String</type>

Setting to configure a set of cores for a task group. There is a placeholder for the highest core id named MAXCOREID to configure independently of the current number of cores. To build more complex expressions a range and a subtraction operator are provided. Example: TaskGroup.2.CoreSet={0..MAXCOREID-1} TaskGroup.3.CoreSet={0,1,2}
